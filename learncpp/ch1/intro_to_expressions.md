## intro to expressions 

### expressions 
- an expression is a non-empty sequence of literals, variables, operators and function calls that calculates a value 
- __evaluation__ 
  - process of executing an expression 
- __return value/result__ 
  - the resulting value produces after evaluating an expression 
- do __not__ end in a semicolon & cannot be compiled by themselves 
- always evaluated as part of statements 
```c++
// five() is a function that returns the value 5
int five() {
    
    return 5;
}

int main() {
    
    int a{ 2 };             // initialize variable a with literal value 2
    int b{ 2 + 3 };         // initialize variable b with computed value 5
    int c{ (2 * 3) + 4 };   // initialize variable c with computed value 10 
    int d{ b };             // initialize variable d with variable value 5
    int e{ five() };        // initialize variable e with function return value 5
    
    return 0;
}
```
- the above initializers all make use of an expression 
- __literals__ evaluate to their own values 
- __variables__ evaluate to the value of the variable 
- __operators__ (such as the operator+) use their operands to evaluate to some other value 
- __function calls__ evaluate to whatever value the function returns 

#### breakdown of a statement 
```c++
int x{ 2 + 3 }; // 2 + 3 is an expression that has no semicolon 
                // the semicolon is at the end of the statement containing the expression 
```
- if we were to break this statement down into its syntax, it would look like this:
```type identifier { expression };```
  - _type_ could be any valid type (int was chosen)
  - _identifier_ could be any valid name (x was chosen) 
  - _expression_ could be any valid expression (2 + 3 was chosen)

### expression statements 
- certain expressions (such as __x=5__) are used primarily for their side effects rather than the value they produce  
  - itc, to assign the value __5__ to the variable __x__ 
- although expressions cannot be executed by themselves, it's trivial to convert any expression into 
  an equivalent __statement__ 
- an __expression statement__ is a statement that consists of an expression followed by a semicolon 
  - when the expression statement is executed, the expression will be evaluated 
- in this way, we can take any expression (such as __x=5__) & turn it into an expression statement (__x=5;__)
  that will compile 
- when an expression is used in an expression statement, __any__ result generated by the expression is discarded 
  because it's not used 

### subexpressions, full expressions & compound expressions 
```c++
2           // 2 is a literal that evaluates to 2
2 + 3       // 2 + 3 uses operator+ to evaluate to value 5
x = 4 + 5   // 4 + 5 evaluates to value 9, which is then assigned to variable x 
```
- __subexpression__ 
  - an expression used as an operand 
    - (ex.) the subexpressions of __x = 4 + 5__ are __x__ and __4 + 5__ 
      - the subexpressions of __4 + 5__ are __4__ and __5__ 
- __full expression__ 
  - an expression that is __not__ a subexpression 
    - (ex.) all three expressions above (2, 2 + 3, x = 4+5) are full expressions 
- __compound expression__ 
  - an expression that contains two or more uses of operators 
    - (ex.) __x = 4 + 5__ is a compound expression b/c it contains two uses of operators (operator= & operator+)
