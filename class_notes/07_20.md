**compile time polymorphism**

- **Polymorphism**
	- when same entity (function or object) behaves different in scenarios 
	- "poly" many + forms 
	- types of polymorphism: 
		- **compile time** 
			- method overloading 
			- operator overloading 
		- **runtime** 
			- virtual function 
			- function overriding 

	- **Runtime**
		- time period where executable code is running 
		- errors can be detected only **after** execution of program 
		- errors are called **runtime** errors 
	- **Compile Time** 
		- time period where code types is converted into executable code 
		- errors detected before execution of the program 
		- errors are called **compile time** errors 
			- semantics error 
			- syntax error 

**Compile-time polymorphism** 
	- function is called at time of program compilation
	- aka **early binding** or **static binding** 
	- (ex.) operator overloading or compile-time polymorphism 

**Overloading** 
	- C++ allows you to specify more than 1 definition for a function name or an operator in same scope 
		- function overloading 
		- operator overloading 

**Function Overloading** 
	- one function can perform many tasks 
	- in C++, single function is used to perform many tasks with same name & diff type of args 
	- readability of program increases by function overloading 
	- achieved by using same name for same action

**Operator Overloading** 
	- can redefine or overload most of built-in operators available in C++ 
		- thus, operators can be used with user-defined types as well 
	- have a return type & parameter list 
	- operators which **CANNOT** be overloaded: 
		1. ?: (conditional) 
		2. . (member selection) 
		3.  .*  (member selection with pointer-to-member) 
		4.  :: (scopre resolution) 
		5.  sizeof (object size info) 
	- operator overloading means defining additional tasks to operators w/o changing its actual meaning 
	- __Purpose__
		- provide special meaning to user-defined data types 
- __Advantage__ 
	- perform different operators on _same_ operand 
- RE: 
	- https://www.tutorialspoint.com/cplusplus/cpp_overloading.htm




**OOP Concepts - Polymorphism** 
	- Run-time polymorphism
	- __function overriding__ is an OOP feature that allows a derived (child) class to provide its own implementation of a **virtual function** that is already defined in the base (parent) class 
	- lets a subclass **replace or redefine** a behavior inherited from its superclass 
	- **key points** 
	- ** virtual functions** 
			- declaring a method as virtual in the base class allows for method overriding in derived classes 
		- **dynamic binding** 
			- correct method is determined at runtime 
		- **memory management** 
			- important to clean up dynamically allocated memory to avoid memory leaks 
	- **same function signature** 
		- overriden function in derived class must have same name, return type & params as one in base class 
- **Virtual keyword** 
	- base class function must be declares as virtual to allow overriding 
**Override** keyword: 
	- optional but recommended 
	- in C++ & later, derived class function can use override keyword to clearly indicate that it overrides a base class function 




** Pure virtual functions & abstract classes** 
	- pure virtual function -> virtual function that has no implementation in the base class & is declared by assigning 0 to it in class definition 
	- indicates that the function must be overriden in any derived class, making the base an **abstract class**
