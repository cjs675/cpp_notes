## private access specifier 
- restricts access of data values to class members 
	- hides access to outside 
- when a class member is __private__: 
	- can only be accessed by methods & friends of that class 
	- this __encapsulation__ helps protect the integrity of the class data by preventing external code from directly accessing or mod'ing internal state of class 
- accessed by getters & set by setters 
- set using methods from within class
- if we have public access, we can access class data values directly from main 


## constructors 
- special 'member function' having same name as that of its class -> used to init some valid values to the data members of an object 
	- same name as class 
	- does not have return type 
	- is __public__ 
- 3 types used in C++: 
	- default 
	- parameterized 
	- copy 
- __special method__ that is auto called when object of a class is created 
- to create, use same name as class, followed by __parentheses__ 

```
class MyClass {	// the class 
	public: 			// access specifier 
	MyClass() {		// Constructor 
		cout << "Hello world";
	}
};

int main() {
	MyClass myObj; // create an object of MyClass (this will call the constructor) 
	return 0;
```


### default constructor syntax 
- class which accepts __no__ parameter 
- if there is no constructor for a class, compiler implicitly creates a default one 
```c++
class class_name {
private: 
.....
.....
public: 
class_name()
	{
	.....
	}
}
```

### parameterized constructor 
- class that can take params 
- used to init objects w/ diff sets of values 

```
Class className {
.....
.....
}

Public:
	className(params) {
	....
	....
	}
};
```

### destructors 
- same class name precedded by __~__ symbol 
- removes & destroys memory of object which constructor allocated during creation of an object 

```
class className {
....
public: 
	xyz(); // constructor
	~xyz(); // destructor
};
```
- has no arg & doesn't return any value so it cannot be overloaded 
- don't have to explicitly call destructors as they're auto called 
- can have __only one__ destructor 



### constructor vs destructor 
- constructor:
	- used to allocate memory to the object
	- may or may not contain args 
	- auto called whenever object of class is created 
	- occupies memory 
	- has  no return types 
	- no special symbol when calling 
	- can be used more than once in program  
- destructor: 
	- used to deallocate the memory that constructor allocated to an object 
	- cannot contains args 
	- destructor releases memory 
	- doesnt have a return type 
	- special symbol (~) 
	- cannot be used more than once 

### constructor vs destructor  
- inheritance 
	- constructor can be inherited 
	- destructor cannot be inherited 
- overloading 
	- constructor can be overloaded 
	- destructor cannot be overloaded 
- execution order 
	- constructors executed in successive order 
	- destructors executed in constructor's __reverse__ order 
		- are __inverse__ of constructors
