### memory management (static vs dynamic) 
- static 
	- allocated: compile time 
	- exists until: 
		- program ends 
	- (ex.) int arr[5];
- dynamic 
	- allocated: runtime 
	- exists until: 
		- explicitly deleted 
	- (ex.) int *arr = new int [5];

#### why dynamic memory? 
- allows memory to be allocated at runtime (instead of compile time) 
- useful when exact amount of memory needed is **not known** in advance such as: 
	- user-defined array sizes 
	- real-time data (sensor readings, input files) 
	- dynamic object creation (linked lists, trees, etc) 

**operators for dynamic memory** 
	- new
	- new[]
	- delete
	- delete[]

### new & delete 
- allocate block of memory: 
	- pointer-variable = **new** data-type[size];
		- where size (a variable) specifies number of elements in an array 

### roles of constructors & destructors 
- constructor: allocates memory dynamically when object is created 
- destructor: frees memory when object goes out of scope 
	- prevents memory leaks 

### common errors
- forgetting to use delete -> **memory leak** 
- using delete on non-dynamically allocated memory ->  **runtime error** 
- using freed memory -> **dangling pointer** 


### differences between new, delete, malloc() & free() 
- memory created by **new** keyword is deleted using the **delete** keyword 
- memory created by **malloc()** is deleted by **free()**
- **new** calls the constructor 
- **delete** calls the destructor for memory deallocation 



### functions 
- aka procedure or subroutine 
- each function characterized by following traits: 
	- name 
	- parameters 
	- types of result to be returned 
- syntax: 
	```cpp
	returnType functionName (param1, param2) {
		// function body 
	}
	```
- types of functions: 
	- **library functions** 
		- built-in functions in cpp
		- programmers can use lib functions by invoking them directly; don't need to be re-written 
	- **user-defined functions** 
		1. functions w/o params w/o return type 
		2. functions w/o params w/o return type 
		3. functions w/ no params w/ return types 
		4. function w/ params w/ return types 

### call by value & call by reference 
- 2 ways to pass value or data to function in C++ 
	- call by value 
	- call by reference 



## inheritance 
- one of key features of OOP 
	- allows class (**derived** class) to inherit properties & behaviors (data members & member functions) from another class (**base** class) 
- purpose: 
	- code reusability
	- extensibility
	- establishing relationships between classes 

- **derived class (sub class)** 
	- class which inherits members of another class 
- **base class** 
	- class whose members are inherited 

- syntax: 
```cpp
class DerivedClassName : accessSpecifier BaseClassName {
	// additional members 
};
```
- **access specifiers:** 
	- public: public & protected members of base class retain their accessibility 
	- protected: public & protected members become protected in the derived class 
	- private: all inherited members become private in derived class 

- inheritance syntax: 
```cpp
class Base {
public: 
	void display() {
		cout << "Base class display" << endl;
	}
};

class Derived : public Base {
	// derived class inherits members of base 
};
```

- **types of inheritance** 
	- single 
	- multiple 
	- multilevel 
	- hierarchical 
	- hybrid 


- **single** 
	- one base - one derived 
		- A -> B
- **multiple** 
	- one derived, multiple bases 
		- A, B, -> C
- **multilevel** 
	- derived class becomes base for another 
		- A -> B -> C
- **hierarchical** 
	- one base -> multiple derived 
		- A -> B, C
- **hybrid** 
	- combination of more than one type 
		- combo of above 
		- A -> B -> C, D


** derived class ** 
- defined as class derived from base class 
```cpp
class derived_class_name :: visibility::mode base_class_name {
	// body of derived class 
}
```
** multiple inheritance ** 
- process of deriving new class that inherits attributed from two or more classes 

** multilevel ** 
- process of deriving a class from another derived class 
	- when one class inherits another class which is further inherited by another class 

** hierarchical inheritance ** 
- defined as process of deriving more than one class from a base class 

** hybrid inheritance ** 
- combination of more than one type of inheritance
